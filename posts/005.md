# Deploy e Produto

Escrever código que só roda no seu computador não é suficiente. Este level cobre o que você precisa para publicar um projeto real e acessível para qualquer pessoa.

---

## GitHub Pages

GitHub Pages é uma forma gratuita de hospedar sites estáticos diretamente de um repositório.

**Requisitos:**
- Repositório público no GitHub
- Arquivo `index.html` na raiz do projeto

**Passos para ativar:**

1. Acesse o repositório no GitHub
2. Vá em **Settings → Pages**
3. Em *Source*, selecione a branch `main` e a pasta `/ (root)`
4. Clique em **Save**

Em alguns minutos o site estará disponível em:

```
https://seu-usuario.github.io/nome-do-repositorio
```

## Estrutura de projeto publicável

Um projeto pronto para deploy tem uma estrutura clara e sem dependências locais:

```
projeto/
├── index.html
├── style.css
├── 404.html
├── config/
│   └── config.js
├── src/
│   ├── app.js
│   ├── router.js
│   ├── ui.js
│   └── marked.min.js
├── pages/
│   ├── home.html
│   ├── about.html
│   └── contact.html
├── posts/
│   └── 001.md
└── data/
    └── posts.json
```

Tudo é arquivo estático. Não há servidor, não há banco de dados, não há processo rodando.

## O arquivo 404.html

No GitHub Pages, quando o usuário acessa uma URL inexistente, o servidor retorna o arquivo `404.html` se ele existir.

Para SPAs com hash router, isso raramente é necessário — a navegação acontece no client. Mas é boa prática ter o arquivo para casos de URL digitada diretamente.

```html
<!DOCTYPE html>
<html lang="pt-BR">
<head>
  <meta charset="UTF-8" />
  <title>404</title>
  <link rel="stylesheet" href="/style.css" />
</head>
<body>
  <main>
    <h1>404</h1>
    <p>Página não encontrada.</p>
    <a href="/">← voltar ao início</a>
  </main>
</body>
</html>
```

## Domínio customizado

Para usar um domínio próprio (ex: `seusite.com`) no lugar do `github.io`:

1. Compre o domínio em qualquer registrador (Namecheap, Registro.br, etc.)
2. No painel do registrador, crie um registro DNS do tipo `CNAME` apontando para `seu-usuario.github.io`
3. No GitHub Pages, adicione o domínio no campo *Custom domain*
4. Crie um arquivo chamado `CNAME` na raiz do repositório com apenas o domínio:

```
seusite.com
```

O GitHub Pages ativa HTTPS automaticamente via Let's Encrypt.

## Checklist antes de publicar

- [ ] `index.html` na raiz do projeto
- [ ] Todos os paths relativos (sem `/` absoluto no início, a menos que necessário)
- [ ] Nenhuma dependência local não commitada
- [ ] `data/posts.json` atualizado com os posts reais
- [ ] Arquivos `.md` presentes na pasta `posts/`
- [ ] Testado abrindo `index.html` direto no browser antes do deploy
