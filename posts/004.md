# Arquitetura de SPA

SPA (*Single Page Application*) é uma aplicação web que carrega uma única página HTML e atualiza o conteúdo dinamicamente, sem recarregar o browser.

---

## O problema que a SPA resolve

Em um site tradicional, cada link carrega uma nova página do servidor. Em uma SPA, o JavaScript intercepta a navegação e renderiza o novo conteúdo no mesmo HTML.

O resultado é uma experiência mais rápida e fluida para o usuário.

---

## Módulos ES6

Antes de construir uma SPA, é importante organizar o código em módulos. Módulos ES6 permitem separar responsabilidades em arquivos diferentes e importar apenas o que for necessário.

```js
// math.js
export function somar(a, b) {
  return a + b;
}

export const PI = 3.14159;
```

```js
// app.js
import { somar, PI } from "./math.js";

console.log(somar(2, 3)); // 5
console.log(PI);          // 3.14159
```

Para usar módulos no browser, o script precisa do atributo `type="module"`:

```html
<script type="module" src="src/app.js"></script>
```

## Hash Router manual

A forma mais simples de fazer roteamento em uma SPA estática é usando o hash da URL (`#`).

Quando a URL muda de `/#/` para `/#/sobre`, o browser não faz nenhuma requisição ao servidor. O JavaScript lê o hash e decide o que renderizar.

```js
// router.js
const rotas = {};

function registrar(caminho, handler) {
  rotas[caminho] = handler;
}

function resolver() {
  const hash = window.location.hash || "#/";
  const caminho = hash.slice(1) || "/";
  const handler = rotas[caminho] || rotas["/404"];
  if (handler) handler();
}

function init() {
  window.addEventListener("hashchange", resolver);
  resolver();
}

export default { registrar, init };
```

```js
// app.js
import router from "./router.js";

router.registrar("/", () => {
  document.querySelector("#app").innerHTML = "<h1>Home</h1>";
});

router.registrar("/sobre", () => {
  document.querySelector("#app").innerHTML = "<h1>Sobre</h1>";
});

router.init();
```

## Separação de responsabilidades

Uma SPA bem organizada divide o código em camadas:

| Arquivo | Responsabilidade |
|---|---|
| `config.js` | configurações globais (paths, nome do site) |
| `router.js` | mapeamento de rotas e navegação |
| `ui.js` | renderização de componentes fixos (navbar, footer) |
| `app.js` | lógica de negócio e orquestração |

Cada arquivo exporta um objeto ou funções. O `app.js` importa e conecta tudo.

## Parâmetros de rota

Para rotas dinâmicas como `/p/:slug`, basta verificar o prefixo:

```js
function resolver() {
  const hash = window.location.hash || "#/";
  const caminho = hash.slice(1) || "/";

  if (caminho.startsWith("/p/")) {
    const slug = caminho.replace("/p/", "");
    rotas["/p/:slug"](slug);
    return;
  }

  const handler = rotas[caminho] || rotas["/404"];
  if (handler) handler();
}
```
